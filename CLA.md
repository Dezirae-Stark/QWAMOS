## 11. How to Agree to This CLA

You indicate Your acceptance of this CLA by taking one of the following actions:

### Option 1: Signed-off-by Commits (Recommended)

Add a `Signed-off-by` line to Your Git commit messages:

```bash
git commit -s -m "Add post-quantum encryption module

Signed-off-by: Your Name <your.email@example.com>"
```

The `Signed-off-by` line certifies that You have read and agree to the terms of this CLA, similar to the [Developer Certificate of Origin (DCO)](https://developercertificate.org/).

### Option 2: Explicit Confirmation in Pull Requests

When submitting a pull request, include the following statement in the PR description:

```
I have read and agree to the QWAMOS Contributor License Agreement (CLA.md).
```

### Option 3: Separate CLA Signature (For Major Contributors)

For contributors making substantial contributions (e.g., entire subsystems, major features), the Project Maintainer may request a separately signed CLA document. In such cases:

1. The maintainer will send You a CLA signature request via email
2. You will sign the CLA digitally (e.g., via GPG signature or DocuSign)
3. The signed CLA will be stored in the `legal/clas/` directory (not public)
